GitHub launch package · 4×4×4 minimum-link covering trail

smart-search-18-order-from-cover64-stitch

Интерактивная памятка к запуску: превращаем найденные в search-17 unordered 64/64 scaffold-линии в настоящую упорядоченную 22-звенную ломаную с реальными точками стыка.

Гипотезаcontact-aware ordering
Источникsearch-17 scaffolds
Масштаб20 shards × 4 workers
Цельordered trail > 60/64

Что это за запуск

Это launch package для шага после smart-search-17-cover64-stitch-graph. Search-17 нашёл наборы из 22 линий, которые как unordered scaffold покрывают все 64 точки. Search-18 пытается собрать из них настоящую ломаную: выбрать порядок линий, контактные вершины и реальные сегменты vertices2.

Коротко: раньше мы нашли хорошие дороги; теперь ищем один маршрут по этим дорогам, не теряя покрытие из-за плохих перекрёстков.

Было

64/64 line-set scaffold, stitch path 22/22.

Проблема

Граф пересечений не равен настоящей ломаной: стык может отрезать полезную часть линии.

Теперь

Оптимизируем actual ordered-chain coverage, а не только unordered line-set coverage.

Почему это нужно

Набор линий может покрывать 64/64, но настоящая ломаная требует, чтобы каждое звено начиналось в конце предыдущего. Если контактная точка лежит внутри богатой линии, линия укорачивается и теряет grid-точки.

Главная ловушка: line-set scaffold может выглядеть полностью сшиваемым, но после выбора реальных контактов покрытие падает.
