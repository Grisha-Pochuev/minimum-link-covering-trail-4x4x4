# Ночной поиск 22-звенной ломаной

Этот запуск не доказывает оптимальность. Он пытается найти 22-звенную цепь или хотя бы хорошие почти-решения.

## Как запускать

Откройте вкладку `Actions`, выберите `overnight-22-search`, нажмите `Run workflow`.

Оставьте значения по умолчанию:

```text
seconds = 20400
box_min = -3
box_max = 7
top_k = 32
```

`20400` секунд это 5 часов 40 минут на каждый кусок. Это сделано специально, чтобы не упереться прямо в шестичасовой предел одного задания.

Запуск делится на 16 кусков:

```text
overnigth-22-shard-0
...
overnigth-22-shard-15
```

После завершения надо открыть конкретный запуск и скачать `Artifacts`.

## Что значит хороший результат

В каждом JSON-файле ищите:

```text
covered_count
missing
vertices
```

Полное 22-звенное решение выглядит так:

```text
covered_count = 64
missing = []
links = 22
```

Если найдено только 62 или 63 точки из 64, это всё равно полезно. Нужно сравнить недостающие точки между кусками. Повторяющиеся наборы недостающих точек показывают, где находится настоящий дефект.
